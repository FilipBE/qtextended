Build system changes for 4.1.4

Overview

This document describes the differences between the 4.1.4 and the 2.x build systems. As the number of changes is very large, this is mostly an overview.

Changes

  • Manually running qmake is no longer supported. All app development must be enabled via configure or qtopiamake.
  • Dependency info is obtained from the project you depend on (much easier to maintain and we get a dependency overview for free).
  • config.pri and global.pri have been split up into many pieces (due to qmake's support for features - implicitly loaded .prf files).
  • Environment variables no longer need to be set (except for QPEDIR, which is required when running scripts).

See also What's New?.

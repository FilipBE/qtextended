Composition of MMS messages

This feature is provided as part of the Touchscreen Phone Reference Design.

MMS messages containing various media types can be composed. A message can consist of a single item of media, or multiple items arranged as an MMS SMIL slide-show of arbitrary length. SMIL slide-shows support the composition of text, image and audio elements, with configurable parameters such as slide duration and background colors.

The following writing-related features are provided:

  • Fields: it is possible to specify a subject, and To/Cc/Bcc recipients, either by phone number or by selection from the Contacts listing. It is possible to request delivery report and read reply.
  • Composing: a standard slide-show with SMIL presentation is supported by the MMS composer. One image, text field and audio file are allowed per page of the slide-show. Slide background color and display duration can be specified, applying to all slides in the slide-show.
  • Modify: it is possible to make changes as necessary to mails stored in the Drafts folder, and either send the modified message or save it back to the Drafts folder.
  • Prompt for save: when a message has been modified in the composer, the user is prompted to save changes on cancellation.
  • Store: draft mails are stored in the Drafts folder and mails ready to send are stored in the Outbox folder, while transmitted messages are stored in the Sent folder. It is possible to move messages from the Outbox folder to the Drafts folder to cancel transmission, and permit further editing.

Reply to an MMS

The MMS viewer enables the user to easily reply to the currently viewed MMS message.

Retrieving MMS Messages

The user is notified of the arrival of an MMS message, with an audible alert. The user is presented with the option of inspecting the message immediately, or deferring viewing until a later time. If multiple new messages have arrived, the Inbox is displayed in preference to any individual message.

The Messages application can be configured to automatically download the body of a message, or to present the user with summary information, and the opportunity to download the body when suitable. An MMS message can be deleted from the Messages application without first downloading the body.

Standards Compliance

Qt Extended MMS provides conformance with OMA MMS 1.2 for the Image Basic content class.

Transports

A transport abstraction layer allows integration with 3rd party WAP stacks. A sample implementation using HTTP over TCP/IP is included as a sample.

Viewing and sending video MMS messages

This feature is provided as part of the Touchscreen Phone Reference Design.

Messages provides support for viewing and sending MMS messages containing video media.

Viewing MMS Messages

This feature is provided as part of the Touchscreen Phone Reference Design.

The MMS viewer is able to render MMS messages containing "MMS SMIL" - a subset of "SMIL Basic" (see http://member.openmobilealliance.org/ftp/Public_documents/MWG/MMS/Permanent_documents/OMA-MMS-CONF-V1_2-20040219-C.zip), containing the elements details below.

The following features are provided:

  • Display of images and text and playing sounds.
  • Playing MMS messages containing SMIL presentation.
  • Reply: it is possible to reply to or forward the currently viewed message.
  • Delete: it is possible to delete the currently viewed message

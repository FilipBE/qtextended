Qtopia Sync Agent changes for 4.3.3

Overview

This page describes changes to Qtopia Sync Agent 4.3.3.

  • Fixed some issues that resulted in real serial connections (not Greenphone USB-serial) being unreliable.
  • Fix a logic bug so that Synchronization will send Calendar events to Qtopia Sync Agent.
  • MAPI has 2 fields for each email address. One contains the email address, the other contains something Exchange-specific. Stop sending the Exchange-specific value to Qtopia (send the email address instead).
  • Contacts created from Exchange addresses have the correct emails transfered to Qtopia.
  • Appointments with removed occurrences in Outlook no longer result in blank events being created in Qtopia.
  • Appointments with exceptions in Outlook no longer result in the Outlook security dialog being displayed.

Known issues

  • None.

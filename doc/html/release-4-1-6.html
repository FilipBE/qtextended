Qtopia - 4.1.6 Release Notes

  • Introduction
  • Enhancements
  • Bug Fixes
  • Known Issues

Introduction

Qtopia 4.1.6 is a patch release that provides:

  • improved performance
  • fixes for a number of bugs
  • many optimizations for the Greenphone.

Enhancements

  1. Performance:
    • Enabled symbol hiding (for the greenphone) to reduce the symbol count of libQt*, which has a small improvement to the time it takes to run applications.
    • Qtopia now supports GCC 4.1.1 which considerably improves performance.

Bug Fixes

  1. Qt
    • BUG : Fixed security issue in qimage.cpp. This makes the Qt version used for this release Qt 4.1.12 + security patch.
  2. Contacts
    • BUG : Changed the text for adding a photo to a contact from "Move to Contact..." to "Save to Contact...". Also changed the icon to use the correct icon for contacts.
    • BUG : Fixed a bug where only one contact would be shown.
    • BUG 134212: Show all contacts except those from SIM when assigning a photo to a contact.
    • BUG : Fixed a crash bug in iAbEditor.
    • BUG 132414: Prompt the user to choose if a new contact is to be created when adding phone numbers from call history to contacts.
    • BUG 132421: A contact with multiple phone numbers did not show the first phone number in Add to Speed Dial on the menu.
    • 132970, 132542: Added support for adding descriptive text to a category selection dialog.
  3. Calendar
    • BUG 132401: In month view it was impossible to navigate around the calendar
  4. Documents API
    • BUG : Check validity of files on removable media to handle the case where an SD card is removed.
    • BUG : Made scanPath "all" scan for applications also.
    • BUG 134030: Moving a document stripped the categories.
  5. Telephony
    • BUG 132385: Automatically switch to hands-free mode when headphones are plugged in.
    • BUG : If a contact has more than one phone number, ask the user to select which phone number they want to send and SMS to rather than using the default number.
    • BUG 134444: DTMF tones did not work on Greenphone.
    • BUG 133610: Prevent duplicate calls being shown when only one is received.
    • BUG 132573: signal strength should not be shown in airplane mode.
    • BUG 132397: Added a message box to indicate when joins and holds fail.
  6. Qtopia Server
    • BUG : QtopiaApplication should not be setting the volume via OSS in phone edition.
    • BUG 132571: Improved heuristics for screen brightness.
    • BUG 132556: ensure the correct value is used after changing the TZ environment variable.
    • BUG 130424: Fixed a crash during long key presses when no Helix is available.
  7. Camera
    • BUG : Fixed a crash that occured when a view no longer owns the model.
    • BUG : Fixed camera issues in Greenphone device.
  8. Full Screen Handwriting
    • BUG : Smoothe mouse input to improve handwriting.
  9. Messages
    • BUG : Prevent messages from closing when hiding the message view in Touchscreen mode.
    • BUG 128610: Improved character limit for SMS messages.
    • BUG 124864: MMS messages with no subject where discarded.
    • BUG 132545: There was no delete all functionality.
    • BUG 132543: Show the contact selector widget in appropriate cases when the select key is pressed.
    • BUG : Fixed a crash when canceling composition.
    • BUG 128614: Use assigned MMS account instead of default so delivery reports are sent properly.
    • BUG 128619: MMS read-reply flag was not honored when choosing to read later.
    • BUG 128612: Prevent re-ordering of multiple accounts when downloading mails.
    • BUG 126510: Don't remove bcc addresses for MMS.
    • BUG 124571: Fixed a problem with decoding of charsets in WSP encoded-text-strings.
  10. Input Methods
    • BUG 134216, 123700: fixed disappearing cursor.
    • BUG 134215: When text was selected, the copy action first copied the text and then removed it.
    • BUG 134047: Touchscreen wasn't completely disabled in keypad lock mode.
  11. Pictures
    • BUG 133609: First image in document list flashed on-screen before the selected image was loaded.
    • BUG 134238: Removed legacy manual double buffering to improve navigation in Pictures.
  12. Appearance
    • BUG 134232: Only change theme on explicit Yes.
  13. Phone Networks
    • BUG 134019: show a wait widget while reading network information.
  14. Call Options
    • BUG 132562: Caller ID blocking didn't work until after the first time it was changed.
  15. Call History
    • BUG 132410: The top item did not have focus when viewing the call details.
  16. Call Forwarding
    • BUG 132558, 132559: Added a To voice mail option to number selector.
  17. World Time
    • BUG 134220: Ensure that QTimeZoneWidget displays the current time zone when selecting "More..."
    • BUG 134219: Canceling "more..." now reverts time zone back to the previous time zone in QTimeZoneWidget
    • BUG 132556: Ensure correct city is selected when cities are closely spaced
    • BUG 132555: Fixed zooming in Worldmap
    • BUG 132554: Improved user interface.
  18. Date & Time
    • BUG 133815: Could not use keypad entries to enter minutes.
    • BUG 134029: Made selecting time zones in date/time more sensible.
    • BUG 133187: Made sure the date is shown when clock first appears (not 0.5 seconds later)
    • BUG 132538: Use select key to start/stop stopwatch.
  19. Voice Notes
    • BUG 133192: Improved suspension mechanism to allow replaying audio files in full, even when the screensaver kicks in.
  20. Handwriting
    • BUG 132567: Drawing animation for the first character in the list was only displayed once.
  21. Tasks
    • BUG 132549: After returning from the edit task screen a task could not be deleted until the user fully returned to the task list screen.
  22. Network
    • BUG 126516: encrypt password when shown on UI

Known Issues

    • Bluetooth is currently not supported on the Greenphone device.
    • Only wav file support is available on the Greenphone (no video playback).
    • MMS and WAP are not currently supported.
    • The user cannot add custom strokes to handwriting.
    • When an SMS arrives the SMS that is opened initially is not the just received SMS.
